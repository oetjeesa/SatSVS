# Example antenna patterns (GRASP formats)

Antenna far-field patterns in the TICRA GRASP file formats, for the com_*
link-budget analyses via `<TransmitAntennaPatternFile>` and
`<ReceiveAntennaPatternFile>` (see the Antenna patterns section of the main
readme):

- `isoflux_x_band.cut` — shaped isoflux LEO X-band downlink antenna
  (3 dBi at nadir rising to 6.5 dBi at 58 deg off-nadir, steep roll-off
  beyond the edge of coverage). Typical for LEO payload downlink.
- `dish_x_band_3m.grd` — X-band 3 m ground station dish at 8.025 GHz
  (~46 dBi), as a GRASP uv-grid file.
- `dish_ka_band_25cm.cut` — Ka-band 0.25 m satellite downlink dish at
  26.25 GHz (~34 dBi).
- `dish_ka_band_6_8m.cut` — Ka-band 6.8 m ground station dish at 26.25 GHz
  (~63 dBi).

The files are generated by `make_patterns.py` (run once with
`py input/example_antenna_patterns/make_patterns.py`) from realistic analytic
models: a uniform circular aperture (Airy) pattern with the ITU-R
32-25log(theta) sidelobe envelope for the dishes, and a shaped isoflux model
for the LEO downlink antenna. They use the standard GRASP far-field power
normalisation (gain dBi = 10log10 of the summed squared field components) —
patterns exported from GRASP or compatible tools with that normalisation can
be dropped in directly.
